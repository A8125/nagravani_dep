When a complaint is submitted, the backend:

Calls Ollama (nomic-embed-text) to generate a 768-dimension embedding of title + description

Queries the DB for existing complaints with the same category and ward that have an embedding

Calculates cosine similarity between the new embedding and each existing one

If similarity > 0.85 → marks as duplicate, increments upvoteCount on the original, recalculates priorityScore = upvoteCount * 10 + daysSinceCreated * 2, returns duplicate: true

If no duplicate → creates new complaint, stores embedding, returns duplicate: false
