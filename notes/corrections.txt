1a. Add a complaint counter sequence
Create a PostgreSQL sequence to auto-increment complaint numbers per department, so CMP-PWD-0001, CMP-PWD-0002, etc. are generated reliably without race conditions.
1b. Alter the complaints table

Change id from VARCHAR(36) (UUID) to a readable string like CMP-PWD-0001
Add citizen_name VARCHAR(255) NOT NULL
Add aadhaar_hash VARCHAR(64) NOT NULL — store a SHA-256 hash of the Aadhaar number, never the raw number itself (legal and privacy requirement)
Add aadhaar_last4 CHAR(4) — last 4 digits only, for display purposes

1c. Update the problems table FK
Since complaints.id is changing format, update the problems → complaints foreign key reference accordingly.

Phase 2 — ID Generation Logic (Backend)
2a. ID format: CMP-<DEPT_SHORT>-<ZERO_PADDED_NUMBER>
Example: CMP-PWD-0001, CMP-CMC-0042
2b. Counter logic in routes/complaints.js

Create a complaint_counters table: (department_short VARCHAR PK, number_of_complaints INTEGER DEFAULT 0)
On each new complaint submission, do an atomic UPDATE complaint_counters SET number_of_complaints = number_of_complaints + 1 WHERE department_short = $1 RETURNING number_of_complaints inside a transaction
Format: CMP-${deptShort}-${String(lastNumber).padStart(4, '0')}
Merged/duplicate complaints also get their own unique ID (they're still separate complaint records)
